# Security Policy

This project is an evaluation framework for narrow scientific AI safety. Please
use coordinated disclosure for issues that could materially change the
responsible-use profile of the repository or dataset.

## Report Privately

Open a private GitHub security advisory:

https://github.com/jang1563/narrow-model-safety-eval/security/advisories/new

For time-sensitive concerns, email `jak4013@med.cornell.edu` with the subject
`NARROW-MODEL SAFETY`.

## Do Not Post Publicly

Please do not include operational sequence details, synthesis routes,
expression vectors, wet-lab procedures, or unreleased model-generated design
outputs in public issues, discussions, pull requests, or comments.

## Scope

Security reports are especially useful for:

- A metric or script that could be reused as a design objective rather than an
  evaluation measurement
- A release-surface gap that exposes withheld generated artifacts
- A stale accession, annotation, or result that changes safety interpretation
- A dependency or workflow issue that could compromise repository integrity

General scientific disagreements, feature requests, and documentation fixes can
use normal GitHub issues.
